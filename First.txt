In a project we will have multiple file where we will write js code, a file is called a module in js
so if we want to execute another modules code/program into another module that we can do it by calling/executing 
that module by the means of require('file path')

Now when we use require to execute a module, it will execute that module but we can't use variables
or methods of that module inside another module, as the members in the module are protected and prevents from leaking
and logically thinking too it should be like that too if i have a module and inside their i use some variables
and do some operations around it and i use that module inside another module where same naming of variables took place then
their would be errors

so if we want to use fuctions/variables of another module we need to export and inside that module where we are using 
it the require will return all the exports of that module and we can import from the require 

var a =10;
function xyz(){
    //does something
}

module.exports ={ a:a,xyz:xyz}

const{a,xyz} = require('module path');

This is also know commonjs export which is by default in node
