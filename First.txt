                        Node.js - JS on Server
Server - A remote computer, machine , CPU which takes request and does some computation and response to that request 
Earlier it was js in the frontend and in backend their were java, php. but after nodejs we can run js on server

V8 - its a js engine build using c++ , it executes js and can be embedded in any c++ applications

EcmaScript - it is a standard/rules that is being used by many scripting languages such as javaScript , and also js engine must follow EcmaScript standard

So if V8 can executes js then why not just put v8 on the server and use it , why the need to nodejs ?
Inside Node their is v8 which is only use to execute js but node also provide api on server, set of fuctions such as fetching 
data from the database etc, which makes node js super powerful

Node.js = Runtime Environment for JavaScript built on Chrome V8 engine

Meaning:

It takes JavaScript code
Uses V8 engine (from Chrome) to run it
Adds extra features like file system, network, server, etc.
     
So Node.js gives JavaScript superpowers outside browser.
