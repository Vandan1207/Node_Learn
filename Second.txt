Express.js
it is a web framework used to create webserver, it is easy to create fast, less redundant code 

we can install express -> npm i express

package.json -> meta data about the project, what dependencies the project is using
example ->
    express:"^4.19.3"
    3-> patch
    19-> minor (backward compatible)
    4-> major (may have major changes, now their could be changes affecting our project)
    ^ -> upgrade to minor and patch changes automatically
    ~ -> patch updates only

Syntax for creating server using express

const express = require("express");
const app = express();

app.use("/ooo",(req,res)=>{
    res.send("listening on ooo path")
})

app.listen(7777,()=>{
    console.log("server is running")
})

app.get(path,(req,res,next)=>{
    // this is known as request handler function
    next()
},()=>{
    // we can handle multiple handler function
})

next -> it will move the execution to next handler function 

** if their is no send() which is use to send response back to the request , and if we not write it than the request will 
run infinity and broswer will terminate it after a time a time interval

Theory 
whenever a request is coming then express will check all the function and checks the matching url, and the one function
which send the response is known as handler function, and then express doesn't move forward, and all the function in between 
is known as middileware

middileware is a function that runs between a request and response and can be used in many ways like authorization, business logic etc..
